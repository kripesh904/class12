Unit 1: Database Management System

Data, Information, Features of Information

Data refers to raw facts and figures without context. Examples: numbers, text, dates.

Information is processed, meaningful data that aids decision making.

Features of Information

  • Accuracy: Information must be correct and free of errors.
  • Relevance: Information should be applicable to the problem at hand.
  • Timeliness: Information must be available when needed.
  • Completeness: Information should be complete without missing parts.
  • Understandability: Information should be clear and easy to interpret.

Database and its Purpose

A database is a structured collection of related data stored electronically.

Purpose: To store data efficiently, reduce redundancy, ensure data integrity, and enable easy retrieval and manipulation.

Terminologies in Database

  • Table: Collection of related data organized in rows and columns.
  • Field: A single attribute or column in a table.
  • Record: A single row in a table representing one entity.
  • Tuple: Another name for a record in relational databases.
  • Object: Data with associated behavior in object-oriented databases.
  • Keys: Attributes used to uniquely identify records (e.g., Primary Key, Foreign Key).

Data Types

Data types define the kind of data a field can hold.

  • Integer: Whole numbers.
  • Float/Double: Decimal numbers.
  • Char: Single characters.
  • Varchar: Variable-length strings.
  • Date/Time: Dates or timestamps.

Data Dictionary

A repository that stores metadata about database objects (tables, columns, constraints).

It helps the DBMS manage the database structure and enforces integrity constraints.

Database Management System (DBMS)

Introduction

DBMS is software that manages databases and provides an interface between users/applications and the data.

Objectives

  • Efficient data storage and retrieval.
  • Data integrity and security.
  • Concurrent access for multiple users.
  • Backup and recovery management.

Advantages

  • Reduces data redundancy.
  • Improves data sharing.
  • Enforces data integrity.
  • Provides data security.

Disadvantages

  • High initial cost.
  • Complex to design and maintain.
  • Requires specialized personnel.

Types of Database Models

Hierarchical Model

Data is organized in a tree-like structure with one-to-many relationships.

Network Model

Data organized as a graph supporting many-to-many relationships.

Relational Model

Data stored in tables with rows and columns, related by keys.

Entity-Relationship Model

Conceptual design using entities, attributes, and relationships to model real-world objects.

Integrity Constraints and Types

Domain Constraints

Specify the permissible values for a field (data type, range).

Entity Integrity

Primary key must be unique and not null.

Referential Integrity

Foreign keys must correctly reference primary keys in related tables.

Key Constraints

Enforce uniqueness and identify records.

Normalization

Introduction

Process to organize tables to reduce redundancy and dependency.

Normal Forms

  • 1NF: Atomic values, no repeating groups.
  • 2NF: 1NF + all non-key attributes fully dependent on primary key.
  • 3NF: 2NF + no transitive dependency.

Advantages

  • Reduces data redundancy.
  • Improves data integrity.
  • Simplifies database maintenance.

Disadvantages

  • May require complex queries.
  • Multiple table joins can reduce performance.

Centralized and Distributed Database

Introduction

Centralized Database: Stored and managed at a single physical location.

Distributed Database: Data is distributed across multiple locations connected via a network.

Advantages of Centralized Database

  • Simple management.
  • Data consistency.
  • Centralized security.

Disadvantages of Centralized Database

  • Single point of failure.
  • Slower access for remote users.
  • Scalability issues.

Advantages of Distributed Database

  • Improved reliability.
  • Faster local access.
  • Better scalability.

Disadvantages of Distributed Database

  • Complex management.
  • Maintaining consistency is harder.
  • Higher implementation cost.

Database Security

Introduction

Database security protects data from unauthorized access, misuse, or threats.

Challenges

  • Unauthorized access
  • SQL Injection
  • Data theft
  • Insider threats
  • Physical damage

Security Measures

  • User authentication
  • Access control and permissions
  • Encryption of sensitive data
  • Firewalls and intrusion detection
  • Regular backups and audits

DBA and Roles of DBA

The Database Administrator (DBA) is responsible for the overall management and maintenance of the database.

  • Managing database security.
  • Performance tuning.
  • Backup and recovery.
  • User management and authorization.
  • Ensuring data integrity and availability.

ER Diagrams

Introduction

Entity-Relationship (ER) diagrams visually represent entities, their attributes, and relationships in a database.

Components

  • Entity: Object or thing (represented by a rectangle).
  • Attributes: Properties of an entity (represented by ovals).
  • Relationships: Associations between entities (represented by diamonds).
  • Cardinality: Specifies number of instances in relationship (1:1, 1:N, M:N).
  • Degree: Number of entities involved in a relationship.

Common Symbols

  • Rectangle - Entity
  • Oval - Attribute
  • Diamond - Relationship
  • Double oval - Multivalued attribute
  • Double diamond - Weak relationship
  • Underline attribute - Primary key

SQL (DDL and DML)

Data Definition Language (DDL)

Commands used to define database schema.


CREATE TABLE Students (
    ID INT PRIMARY KEY,
    Name VARCHAR(50),
    Age INT
);

ALTER TABLE Students ADD COLUMN Email VARCHAR(100);

DROP TABLE Students;

Data Manipulation Language (DML)

Commands used to manipulate data.


INSERT INTO Students (ID, Name, Age) VALUES (1, 'Alice', 20);

UPDATE Students SET Age = 21 WHERE ID = 1;

DELETE FROM Students WHERE ID = 1;

SELECT * FROM Students;

Data Constraints

  • NOT NULL: Field must have a value.
  • UNIQUE: Field value must be unique.
  • PRIMARY KEY: Uniquely identifies each record.
  • AUTO_INCREMENT: Automatically increments numeric value.

Operators

  • AND: Both conditions must be true.
  • OR: At least one condition must be true.
  • NOT: Negates a condition.

WHERE Clause

Used to filter records.


SELECT * FROM Students WHERE Age > 18 AND Name LIKE 'A%';

ORDER BY Clause

Sorts the result set.


SELECT * FROM Students ORDER BY Name ASC;

SQL JOINs

Used to combine rows from two or more tables.

INNER JOIN


SELECT Students.Name, Courses.CourseName
FROM Students
INNER JOIN Enrollments ON Students.ID = Enrollments.StudentID
INNER JOIN Courses ON Enrollments.CourseID = Courses.ID;

LEFT JOIN


SELECT Students.Name, Courses.CourseName
FROM Students
LEFT JOIN Enrollments ON Students.ID = Enrollments.StudentID
LEFT JOIN Courses ON Enrollments.CourseID = Courses.ID;

RIGHT JOIN


SELECT Students.Name, Courses.CourseName
FROM Students
RIGHT JOIN Enrollments ON Students.ID = Enrollments.StudentID
RIGHT JOIN Courses ON Enrollments.CourseID = Courses.ID;

FULL JOIN


SELECT Students.Name, Courses.CourseName
FROM Students
FULL OUTER JOIN Enrollments ON Students.ID = Enrollments.StudentID
FULL OUTER JOIN Courses ON Enrollments.CourseID = Courses.ID;
